# WinrSwipe — App Store Preparation Design

**Date:** 2026-03-22
**Status:** Approved
**Platforms:** iOS + Android

## Overview

Prepare WinrSwipe for submission to both the Apple App Store and Google Play Store. The work covers four areas: icon asset pipeline, configuration updates, store listing content, and screenshot guidance.

---

## 1. Icon Asset Pipeline

### Approach
A Node.js script (`scripts/generate-icons.js`) using the `sharp` library generates all required icon sizes from a single 1024×1024 SVG master.

### Icon Design (B3 — chosen by user)
- **Background:** #0D0D0D (solid black)
- **Shape:** Rounded rectangle (22px radius at 1024px scale)
- **Element:** Three stacked cards in a fan arrangement — background cards in semi-transparent orange (#FF4D1C at 20% and 40% opacity), front card in solid orange gradient (#FF5A28 → #CC3300)
- **Letter:** Bold W in white, centered on front card
- **Badge:** Small circle (bid arrow ↑) in bottom-right corner, solid #FF4D1C with white arrow, dark border

### How EAS Handles Icon Sizes

EAS Build auto-generates all required iOS and Android icon sizes from the single 1024×1024 PNG at `expo.icon`. The script does **not** need to output individual sizes — it only needs to write the final master icon to `assets/images/icon.png`.

## 2. Configuration Updates

### app.json additions and fixes

**Add:**
```json
{
  "expo": {
    "description": "Tinder-style auctions — swipe, bid, win.",
    "ios": {
      "category": "APP_SHOPPING"
    }
  }
}
```

**Fix — rename "SwipeBid" → "WinrSwipe" in ALL six permission strings:**
```json
"infoPlist": {
  "NSCameraUsageDescription": "WinrSwipe uses your camera to photograph items you want to sell.",
  "NSPhotoLibraryUsageDescription": "WinrSwipe uses your photo library to upload product images.",
  "NSMicrophoneUsageDescription": "WinrSwipe uses the microphone to record short product videos.",
  "NSLocationWhenInUseUsageDescription": "WinrSwipe uses your location to show nearby auctions.",
  "NSUserNotificationsUsageDescription": "WinrSwipe sends notifications when you are outbid or win an auction."
}
```
Also fix in plugin fields:
```json
["expo-camera", { "cameraPermission": "WinrSwipe uses your camera to photograph items for sale." }],
["expo-location", { "locationWhenInUsePermission": "WinrSwipe uses your location to show nearby auctions." }]
```
> ⚠️ These strings appear verbatim on iOS permission dialogs. Submitting with "SwipeBid" while the app name is "WinrSwipe" is a rejection risk under Apple guideline 2.3.7.

**Fix — update slug:**
```json
{ "slug": "winrswipe" }
```

### eas.json — Stripe key placeholders
The file currently contains `"your_stripe_test_key_here"` and `"your_stripe_live_key_here"`. These must be replaced manually by the developer with real Stripe publishable keys before building. They are **not** committed to source control as real values.

### eas.json — Android submit config
Add Android submit block with service account key path:
```json
"submit": {
  "production": {
    "ios": { "appleId": "boaz65sa@gmail.com" },
    "android": {
      "serviceAccountKeyPath": "./google-play-service-account.json",
      "track": "production"
    }
  }
}
```
> ⚠️ The **first** upload to Google Play must be done **manually** via the Play Console — `eas submit` only works for subsequent updates. Also, `google-play-service-account.json` must be downloaded from Google Play Console → Setup → API access and added to `.gitignore`.

### .gitignore additions
```

google-play-service-account.json
.env
```

✓ Swipe through live auctions in real time

### Keywords (iOS — 100 chars max)
`auction,bid,swipe,marketplace,escrow,buy,sell,secondhand,deals,winrswipe`

### Google Play Tags
Shopping · Marketplace · Auction

### Privacy Policy URL
Must be a publicly accessible URL. Suggested: add `app/privacy.tsx` content to a hosted page (e.g., `winrswipe.com/privacy` or a GitHub Pages URL). This is **required** before submission.

### Age Rating
Target: **4+ (iOS) / Everyone (Android)**

**iOS:** Complete the App Store Connect content rating questionnaire carefully:
- "Simulated Gambling" → **No** (this is a legitimate marketplace auction, not gambling)
- "Real Money Gambling, Lotteries" → **No**
- "Real Money Gaming" → **No**
- All other categories → No

Answering incorrectly risks a 17+ rating or rejection. WinrSwipe is a peer-to-peer auction marketplace, legally equivalent to eBay.

**Google Play:** Complete the Financial Services declaration — the app facilitates real-money transactions. Select "Shopping" as primary category. No gambling declarations needed.

### Privacy Policy — Pre-submission Gate
A publicly accessible HTTPS URL to the privacy policy is **required** before submitting to either store. The app already has `app/privacy.tsx` — its content must be hosted at a live URL (e.g., `winrswipe.com/privacy` or a GitHub Pages page) **before** creating the first submission. This is not optional and will block submission if missing.

## 4. Screenshots

### Required Sizes
- **iOS:** 1320×2868px (iPhone 16 Pro Max) — primary required size as of 2025. Also prepare 1284×2778px (6.5" iPhone 14 Pro Max) for older device compatibility. Minimum 3, maximum 10 per size.
- **Android:** 1080×1920px minimum, minimum 2
- **iPad:** Not required — `supportsTablet: false` in `app.json`.

### Recommended 5 Screens (manual capture on simulator/device)
1. **Swipe screen** — main auction feed with swipe animation visible
2. **Search screen** — category filters + GPS nearby listings grid
3. **Listing detail** — product carousel + bid history + timer
4. **Payment screen** — Safe Trade escrow breakdown + pay button
5. **Won screen** — active purchases with escrow status badges

### Process
Run `npx expo start` → open on iPhone 16 Pro Max simulator → take screenshots via `Cmd+S` in simulator or `xcrun simctl io booted screenshot`.

## 5. Build & Submit Commands

```bash
# Install sharp for icon generation
npm install --save-dev sharp

# Generate all icons
node scripts/generate-icons.js

# Build for both platforms
eas build --platform all --profile production

# Submit to stores (after build completes)
eas submit --platform ios
eas submit --platform android
```
